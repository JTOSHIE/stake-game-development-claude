5. PLATFORM REALITY (mined first-hand from stake-engine.com approval docs; mirrored under docs/stake-engine-live/)
Review: three anonymous reviewers score on a fractional scale (0 to 3 in about 0.33 steps); the rounded average is the star tier; an average below 1.0 is not published, the thread locks for 7 days, then resubmission is allowed. Post-approval lockdown: cosmetic updates only, no math, mode or gameplay changes ever. Hard technical rules: the build is static and may reach NO external source (external fonts are the classic failure; Orbitron is self-hosted, and the conformance sweep asserts every request is same-origin); mini-player popout must render undistorted; spacebar maps to the bet button; every bet level from the authenticate response must be usable with minStep respected; per-mode cost, RTP and max win must be displayed to the player in-game along with all symbol pays, special values, feature access descriptions and a UI button guide; multi-win outcomes count up incrementally; sound can be disabled; autoplay needs explicit confirmation; language parameters must not corrupt text; social-mode strings must suit stake.us. The game tile is composed in the dashboard Tile Editor from uploaded layers (background render, transparent foreground hero, gradient, title), and the provider logo is a one-time square upload in Team Settings Branding. Team name, game title and assets must clear IP and trademark law: infringement is grounds for rejection. Released games start at the bottom of New Releases and re-rank every Friday, Australian time.
