## 3. Wincap flow

Per the owner's decision: MaxWinCelebration still shows immediately on a capped round
(unchanged). On COLLECT, the game now presents the complete round sequence through the
interpreter â€” "how it happened" â€” finishing on the total win summary, before autoplay or
the next spin can proceed.

- **`FreeSpinsPresentation.svelte`**: `start()` now also handles a non-triggered script (a
  plain base-game win that reaches the cap without ever entering the feature â€” statistically
  near-impossible in this game's maths, but implemented for generality): it shows the base
  spin's board/wins, then the same total-win summary screen a triggered round ends on.
- **`App.svelte`** (`handleSpin`, `handleBuy`): added `waitForWincapCollect()` /
  `handleWincapCollect()`. When `$isWincap`, the flow now awaits the player's COLLECT click
  before presenting (`presentFeature`) whatever script is available, instead of the
  presentation and the wincap flag being set independently of each other as before.
- **`ReplayMode.svelte`**: gained the same flow (it did not show any wincap splash at all
  before this pass â€” `MaxWinCelebration` is now mounted here for the first time). Both the
  feature-round branch and the plain-base-round branch check `payoutMultiplier >= WINCAP`
  first, show the splash, await collect, then run their existing presentation/reveal step,
  then set the final win/wincap state for the summary. Applies "in normal play, bonus, and
  replay" per the brief.
- **Dev-only test hook**: `serveMockRound`'s sibling `serveCategory` (already existed) is now
  reachable via a `?mockCategory=<name>` URL param in `handleBuy` (DEV-gated), so headless
  verification can force the curated `wincap` sample round deterministically instead of
  relying on random selection.

**Proof â€” headless run against the `bonus/wincap` sample round** (`?mockCategory=wincap`):
splash shown (`wincap-splash.png`, "MAX WIN REACHED! 5,000Ă— BET"), COLLECT clicked, full
presentation played (`wincap-presentation.png` â€” live board, meter, spins remaining, running
total), reaching the end/summary phase (`wincap-summary.png` â€” "FEATURE COMPLETE $2,500.00",
instrument column showing final Multiplier 2Ă— / Total Win $5,000.00). All three screenshots
committed to `reports/screens/ux-v1/`.

## 4. Mock pool expansion

Extended `frontend/src/lib/mock/sample_rounds.json`'s bonus-mode set from 22 to 31 rounds,

  code) now draws from all 31 rounds â€” every bonus round in this game triggers the feature,
